B站抢票工具使用说明
=====================

功能特点
--------
- 自动监控B站大型活动票务状态
- 检测到有票时自动抢票
- 实时显示抢票进度和状态
- 支持多座位优先选择

安装步骤
--------
1. 确保已安装 Python 3.8+
2. 安装依赖包：
   pip install -r requirements.txt

使用方法
--------
1. 运行程序：
   python main.py

2. 登录B站账号：
   - 点击"登录B站"按钮
   - 选择包含B站Cookie的JSON文件
   - Cookie文件格式示例：
     {
       "SESSDATA": "your_sessdata_value",
       "bili_jct": "your_bili_jct_value",
       "DEDEUSERID": "your_user_id"
     }

3. 配置活动信息：
   - 输入目标活动的B站活动ID
   - 可以从B站活动页面URL中获取活动ID

4. 开始抢票：
   - 点击"开始自动抢票"按钮
   - 程序将自动监控票务状态
   - 检测到有票时自动执行抢票

5. 抢票成功后：
   - 程序会提示抢票成功
   - 请在B站APP中完成支付

获取Cookie的方法
---------------
1. 在浏览器中登录B站 (bilibili.com)
2. 按F12打开开发者工具
3. 切换到Network（网络）标签
4. 刷新页面，找到任意一个API请求
5. 在请求头中复制Cookie
6. 使用浏览器扩展或在线工具解析Cookie为JSON格式

注意事项
--------
- 请确保B站账号已绑定手机号和邮箱
- 抢票成功率受网络速度和服务器响应影响
- 建议提前登录并保持账号状态正常
- 请在开票前启动程序进行监控

配置文件说明
-----------
config.py 中可以修改的配置：
- VCTCN_SHANGHAI: 目标活动信息
- GRAB_INTERVAL: 抢票轮询间隔（秒）
- MONITOR_INTERVAL: 监控轮询间隔（秒）
- MAX_RETRY: 最大重试次数
- TIMEOUT: 请求超时时间（秒）

故障排除
--------
1. 登录失败：
   - 检查Cookie是否有效
   - 确认Cookie格式是否正确
   - Cookie可能已过期，需要重新获取

2. 无法抢票：
   - 确认活动是否已开始
   - 检查网络连接是否正常
   - 可能票已售罄

3. 程序无响应：
   - 检查日志文件中的错误信息
   - 重启程序尝试

技术支持
--------
如遇到问题，请检查：
- 程序运行日志（~/.bilibili-ticket/ticket.log）
- Python依赖是否正确安装
- 网络连接是否正常
